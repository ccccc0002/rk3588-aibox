# RK3588 AI Box V1 测试用例清单

> 文档日期：`2026-03-03`  
> 对应文档：`docs/plans/2026-03-03-rk3588-ai-box-v1-PRO.md`  
> 目标：用于研发自测、测试执行、联调验收。

---

## 1. 测试执行说明

1. 用例编号规则：`TC-模块-序号`。
2. 优先级：`P0`（阻塞上线）、`P1`（高优先）、`P2`（常规）。
3. 通过标准：步骤全部符合预期结果。
4. 缺陷标准：任一步骤不符合预期即记录缺陷并关联用例编号。

---

## 2. 基础环境

1. 硬件：RK3588 + 16GB + 256GB eMMC。
2. 输入源：
- 20 路 RTSP/ONVIF 摄像头（含 H.264/H.265 混合）。
- 1 套 GB28181 设备或模拟平台。
- 1 套 GAT1400 对端（南向/北向）。
- MP4 样本文件和图片流样本。
3. 测试账号：
- `tenant_admin`、`project_admin`、`operator`、`auditor`。

---

## 3. 核心功能用例

## 3.1 接入与预览（主码流、无转码）

1. `TC-INGEST-001`（P0）  
标题：RTSP 单路主码流接入成功  
前置：摄像头在线  
步骤：新增 RTSP 设备 -> 保存 -> 启动预览  
预期：设备状态在线，预览正常出图，无转码告警

2. `TC-INGEST-002`（P0）  
标题：20 路同时接入稳定  
前置：20 路摄像头可用  
步骤：批量导入 20 路 -> 全部启用 -> 运行 30 分钟  
预期：在线率 >= 99.5%，无批量掉线

3. `TC-INGEST-003`（P1）  
标题：断网重连恢复  
前置：单路在线  
步骤：断开摄像头网络 60 秒 -> 恢复网络  
预期：系统自动重连，状态恢复在线

4. `TC-PREVIEW-001`（P0）  
标题：1/4/9/16 分屏预览切换  
前置：至少 16 路在线  
步骤：切换 1/4/9/16 分屏  
预期：切换成功，画面显示正确，无崩溃

5. `TC-PREVIEW-002`（P0）  
标题：预览链路无编码器调用  
前置：开启系统调试日志  
步骤：连续预览 10 分钟  
预期：日志中无 encode 路径调用

6. `TC-PREVIEW-003`（P1）  
标题：H.265 终端能力白名单生效  
前置：一台支持 HEVC 浏览器，一台不支持  
步骤：分别登录并打开同路 H.265 预览  
预期：支持端正常播放；不支持端给出明确提示

## 3.2 分析与调度

1. `TC-AI-001`（P0）  
标题：检测结果正常产出  
前置：摄像头含人车目标  
步骤：启用检测规则 -> 查看实时检测结果  
预期：目标框、类别、置信度正常输出

2. `TC-AI-002`（P0）  
标题：自动抽帧正常负载策略  
前置：低压环境  
步骤：运行 10 路并观察调度指标  
预期：多数通道分析 FPS 约为 8

3. `TC-AI-003`（P0）  
标题：自动抽帧高压降帧策略  
前置：高压场景（20 路高目标密度）  
步骤：压测 20 分钟  
预期：普通路降帧到 2~5 FPS，系统不崩溃

4. `TC-AI-004`（P1）  
标题：告警通道升帧优先  
前置：存在告警触发场景  
步骤：触发告警 -> 观察目标路分析 FPS  
预期：目标路临时升帧到策略上限

5. `TC-AI-005`（P1）  
标题：温控触发全局降帧  
前置：温控阈值可模拟  
步骤：模拟温度升高  
预期：系统按比例全局降帧，服务可用

## 3.3 告警、媒体与存储

1. `TC-EVENT-001`（P0）  
标题：告警事件闭环  
前置：规则已配置  
步骤：触发入侵告警  
预期：生成 event_id、关键帧、短视频、数据库记录

2. `TC-EVENT-002`（P0）  
标题：告警短视频默认 20 秒  
前置：默认配置  
步骤：触发事件并下载视频  
预期：视频时长约 20 秒

3. `TC-EVENT-003`（P1）  
标题：前端修改告警视频时长生效  
前置：管理员权限  
步骤：配置改为 12 秒 -> 触发新告警  
预期：新事件视频时长约 12 秒

4. `TC-STORAGE-001`（P0）  
标题：85% 高水位触发清理  
前置：媒体盘占用 >= 85%  
步骤：触发新媒体写入  
预期：自动启动 oldest-first 清理

5. `TC-STORAGE-002`（P0）  
标题：清理至 70% 低水位停止  
前置：正在清理  
步骤：持续观察  
预期：占用降到 <= 70% 后停止清理

6. `TC-STORAGE-003`（P1）  
标题：清理不破坏事件元数据  
前置：存在历史事件台账  
步骤：执行清理后查询历史事件  
预期：元数据可查，媒体缺失有明确状态

1. `TC-PROTO-001`（P0）  
标题：HTTP Webhook 推送成功  
前置：配置可达 URL  
步骤：触发告警

2. `TC-PROTO-002`（P0）  
标题：Webhook 重试与幂等  
前置：对端返回 500  
步骤：触发告警 -> 观察重试  
预期：按退避重试；同 event_id 不重复入库

3. `TC-GAT-001`（P0）  
标题：GAT1400 南向接收映射  
前置：南向平台可发报文  
步骤：发送标准报文

4. `TC-GAT-002`（P0）

标题：GB28181 注册与保活

步骤：设备注册 -> 运行保活

7. `TC-GB-002`（P1）  
标题：GB28181 目录同步  
前置：存在 GB 目录数据  
步骤：执行目录同步

8. `TC-GB-003`（P1）

步骤：触发级联事件流转

## 4. 缺陷分级建议

1. `S1`：阻塞上线（核心流程不可用、数据泄露、严重崩溃）。
2. `S2`：高优先（主要功能异常、明显性能不达标）。
3. `S3`：中优先（功能边缘异常、可绕过）。
4. `S4`：低优先（UI 文案、轻微交互问题）。
